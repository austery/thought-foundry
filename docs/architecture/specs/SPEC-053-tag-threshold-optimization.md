# SPEC-053: Tag Threshold Optimization (Occurrence Filtering)

## 1. Goal

Reduce the number of generated tag pages by ~95% to improve build speed and memory efficiency, without losing the navigation value of popular tags.

## 2. Background

In early 2026, data analysis revealed that 84% of unique tags in the Thought Foundry corpus were "singletons" (appearing in only one article). With over 5,000 unique tags being generated by AI, Eleventy was forced to render 5,000+ individual tag pages. This dominated slug computation costs and increased the final artifact size on GitHub Pages.

## 3. Design Decision

**Chosen approach**: Implement a minimum occurrence threshold for tag page generation. Only render a `/tags/<slug>/` page for tags that appear in **5 or more** articles.

**Rationale**: 
1. **Efficiency**: Reducing tag pages from ~5,000 to ~100 results in a ~30-40% reduction in total build time.
2. **UX Clarity**: A tag page with only one article provides little navigation value beyond a direct search. By filtering for 5+ occurrences, we ensure tag pages represent meaningful clusters of content.
3. **Data Integrity**: The raw tag data in Markdown frontmatter is preserved; only the *rendering* of the listing pages is suppressed.

| Alternative | Pros | Cons | Decision |
|-------------|------|------|----------|
| Render All Tags | Complete transparency | Huge build overhead; cluttered navigation | ❌ Rejected |
| Client-side tag filtering | No build cost | Complexity in frontend JS; SEO impact | ❌ Rejected |
| **Threshold Filtering (C≥5)** | Drastic build speedup; high navigation value | Hides low-frequency tags from site map | ✅ Chosen |

## 4. Implementation phases

### Phase 1: Logic Implementation — ✅ Completed

- [x] Modify the `tagList` collection in `.eleventy.js` to include `.filter(tag => tag.count >= 5)`.
- [x] Ensure `all-tags.njk` (the listing page) only displays tags that pass the threshold.

### Phase 2: Template cleanup — ✅ Completed

- [x] Verified that top navigation and tag clouds correctly handle the reduced set.
- [x] Confirmed that `all-tags.njk` pagination remains functional (though less critical with fewer tags).

## 5. Acceptance Criteria

- [x] Tag pages reduced from ~5,000 to ~100-200.
- [x] Measurable reduction in "Slug Miss" cache events during build.
- [x] Build time reduction of at least 20%.
