## Ray 的核心观点：Context Engineering 与打破思维限制

Ray: 你看我就发现，张敏你把你的那个 prompt 拉下来，让大家可以看看你的那个 prompt。然后这 prompt 是你想了很久之后写出来的，还是你写出来还是前半部分。

张敏: 前半部分是我说的，是我跟他说的，然后穿插着一些我跟 AI 的对话。但后半部分就是从女性生理，从这个周报开始，从这儿开始是用 AI 帮我，就是他是是用的 Claude 帮我优化的 prompt，就是这个核心目标。从这里核心目标，然后核心的功能，这有数据维度场景功能。

Ray: 对，到这儿就也就是说你在给这个 problem 之前，你是先和 Claude AI 先对过话的。你跟他说过我想开发一个什么东西，然后他要帮你一起去想。对，是的，对，这个其实挺，这是一个非常好的一个 practice，尤其像在 Replit 这个地方，你每做一笔要花钱的，我们借用其他的 AI 先去做商量。就是我以前做的时候，就我做那个 dear k 也是类似。到后来就是我两个，就我一个项目开始的时候，我就同时开 2 个 AI，1 个 AI 就 Replit 自己本身的，另外一个就比如说 ChatGPT 或者 Gemini，或者说这个 Claude 那时候还没有用 Claude。然后开开之后从头开始就跟他聊，也就是说我跟 Replit 中间发生了什么，其实他也知道，我其实我就告诉他，我是在用 Replit 的这个 AI agent 在帮我们去做这个 App。也就是说其实这是我最近很大的一个感悟，就是说我们老说 prompt engineering。我最近的感悟就是我们现在用 AI 做事情更多是一个 **Context Engineering** (上下文工程：一种将任务的完整背景信息提供给AI，使其能更精准执行任务的方法)，就是这个 AI 已经足够强大了。你需要做的是把你的 context，你脑子里的这点东西，你要全都怎么想办法告诉他。就好像刚才比如说有人说我问问咱社群里的人或怎么样，或者我自己写什么之类的。No, 我说不，我们应该去问 AI，让 AI 来帮我们做一切。这里面本质它属于是一个 context engineer，是因为是强迫你自己，我不去执行，我去绑定 context，全都告诉 AI。如果你能把你的这些 context，你这些背景信息，甭管乱七八糟，比如说张敏说，我冬天北京跑步的时候，那个空气让我也也很不爽，保不齐他就给你搞一整一个空气质量指数。今天是不是适合跑步？跑步要怎么样？或者几点钟才去跑，对不对？比如说这个东西不一定你能想到这个功能，但是你把你的真实的 context 就像瞎聊一样废话连篇的，你告诉他这个其实都是有意义的。所以这是第一个，我觉得你这个 practice 挺好的。

### 压榨AI：提出“非分”的要求

Ray: 第二个就是我想问你一个问题，就是你刚才说用那个 Gemini CLI 的时候，你不知道要怎么做，就是说 Replit 跟 Gemini CLI 对你来说他们的区别在哪？为什么你觉得我不知道这个 Gemini CLI 怎么做，但是我在 Replit 就同样的 prompt，你有扔给过 Gemini CLI 让他去帮你试一试吗？

张敏: 我有的，我扔给他，他的反应就是给我告诉我应该出每一步应该怎么做，然后给我一堆代码，然后我不知道是拿这些代码上哪儿去实践，就是上哪儿去给我实施出来，去给我开发出来。但是我用 Replit，我只要把这个 prompt 扔给他之后，他给我一个视觉化的东西。我一看好像有点那样了。我可以在这个基础上改，甚至有可能我就直接在这个平台上去做了。我觉得它对于开发来说是更就是它是一个生态都在里边都可以做出来。但是 Gemini CLI 你他不可能在里边运行，他是他必须得再有一些，其实可以的，对这些东西都可以跑到你自己。

Ray: 所以我就问你，你有没有坚持你跟 Gemini CLI 来说，no, 我这个事情要你做，而不是我做，我是要你帮我做。你有没有问过他？

张敏: 我还没有这么使劲压榨他，我不会。

Ray: 你看这就我想说你提的要求不够非分，他以为你在找他要建议。这其实里边 Replit 跟 Gemini CLI 它本质区别就是说这个产品好不好，它本质区别在 Replit，它本身给他自己的 AI 的定义，他这个 system prompt 里面就定义就是说他就是来做软件的，就是给用户做软件的。所以你里边说啥，他的解法都是要给你做软件做出来。但在 CLI，它这个东西可能还没有做好，没有做完。他以为有时候他以为他自己是个聊天机器人，他甚至都不知道他自己在哪住着，他不知道他在你电脑里他能操控你的那个浏览器什么之类的。所以你要不说请你不去压榨他是不行的。所以你要说我要你给我做出来，你给我开发，那你可以试一试，他其实可以。对，好好，对，因为你要注意，你看你这什么，你那个是你这个是 web app，你看你要是了解，你可以问一下 AI 什么叫 web app？你应该可以再问他，我将来我想在手机上也能用，在什么上也能用。然后你又说我就是小白，这玩意我应该做成一个什么样的软件？它有各种各样不同的软件，你可以去问他。然后最后你问他，你跟他说，你从这个专家的角度来说，从和创业者的角度来说，你你要做这个东西的话，你要做什么平台的，你要做什么样的软件，是什么 web app 手机的？比如说苹果上的这个 iPad，它它是有点不一样的。那你要你要问好他，别做到最后，你中间再改是吧？所以其实就是还是 context engineer，你告诉他很多 context，包括就是你自己不懂的这个 context 要告诉他。对，所以你可以回头你再去试试 Gemini CLI，然后你你可以看一看。
